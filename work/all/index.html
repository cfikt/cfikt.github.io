Linux 篇

什么是进程

进程是正在运行程序的实例，是操作系统中的一个基本概念，每个进程都有自己的内存空间、程序代码和数据。操作系统通过管理进程来协调计算机资源的分配和使用

​

​

进程的基本状态

进程的基本状态包括：

  1. 新建状态（Created）：进程被创建但还没有开始执行
  2. 就绪状态（Ready）：进程已经准备好执行，但还没有被分配到 CPU 资源
  3. 运行状态（Running）：进程正在执行，占用 CPU 资源
  4. 阻塞状态（Blocked）：进程因为某种原因暂时无法执行，等待外部事件的发生，如等待输入/输出、等待资源等
  5. 终止状态（Terminated）：进程执行完毕或被操作系统强制终止，进程的资源被彻底释放

​

进程和线程

  • 进程：进程是正在运行程序的实例，是操作系统中的一个基本概念，每个进程都有自己的内存空间、程序代码和数据。操作系统通过管理进程来协调计算机资源的分配和使用

  • 线程：进程划分成的更小的运行单位，是操作系统能够进行运算调度的最小单位

进程和线程都是系统中的执行单位，但是它们的调度、资源占用、通信和安全性等方面有所不同

MySQL 篇

InnoDB 与 MyISAM 的区别

InnoDB 支持事务、行级锁、外键约束等特性，适用于需要大量写入和更新的应用。而 MyISAM 则不支持事务和行级锁，适用于读取频繁的应用

  • InnoDB 支持事务，MyISAM 不支持事务
  • InnoDB 支持外键，MyISAM 不支持外键
  • InnoDB 支持 MVCC(多版本并发控制)，MyISAM 不支持
  • InnoDB 支持表、行级锁，而 MyISAM 支持表级锁
  • InnoDB 表必须有主键，而 MyISAM 可以没有主键
  • InnoDB 存储引擎提供了具有提交、回滚、崩溃恢复能力的事务安全，与 MyISAM 比 InnoDB 写的效率差一些，并且会占用更多的磁盘空间以保留数据和索引

​

​

什么是数据库事务

数据库事务是一组数据库操作，这些操作组成一个工作单元，要么全部执行成功，要么全部失败回滚。事务是确保数据完整性和一致性的一种机制

​

​

事务的四大特性

  1. 原子性（Atomicity）：事务是一个不可分割的工作单元，要么全部执行成功，要么全部执行失败回滚，不会出现部分执行的情况
  2. 一致性（Consistency）：事务执行前后，数据的完整性和一致性必须得到保障，简单来说就是事务执行前后，数据库的状态必须保持一致
  3. 隔离性（Isolation）：多个事务并发执行时，每个事务都应该感觉不到其他事务的存在，即事务之间相互独立，互不干扰
  4. 持久性（Durability）：事务一旦提交，对数据库中的数据的改变将永久保存在数据库中

​

​

数据库的隔离基别

数据库的隔离级别是指在多个事务并发访问数据库时，数据库管理系统为了保证事务的隔离性而采取的一种措施，数据库的隔离级别决定了每个事务所能看到数据的程度和受到影响的程度，其主要目的是避免数据读取和更新操作时出现的问题

​

常见的数据库隔离级别包括：

  1. 读未提交（Read Uncommitted）：事务可以读取其他未提交事务的数据，产生脏读
  2. 读已提交（Read Committed）：事务只能读取已经提交的数据，产生不可重复读
  3. 可重复读（Repeatable Read）：同一个事务内多次查询相同数据时，查询结果一致，产生幻读
  4. 序列化（Serializable）：事务串行执行，避免了所有并发问题，但是会降低系统的并发性能

​

​

脏读、不可重复读与幻读

  • 脏读（Dirty Read）：指一个事务读取了另一个事务未提交的数据时，就出现了脏读。如果这个事务回滚或者修改了数据，那么读取到的数据就是无效的或者错误的
  • 不可重复读（Non-repeatable Read）：同一个事务内，由于其他事务对数据的修改，导致多次读取同一数据时得到的结果不一致
  • 幻读（Phantom Read）：同一个事务内，由于其他事务插入了新的数据或者删除了已有数据，导致多次读取数据范围不一致

​

​

什么是 MySQL 主从复制

MySQL 主从复制是一种数据复制技术，它可以将一个 MySQL 数据库的数据复制到另一个 MySQL 数据库中。其中，主库是数据源，从库是数据目标，主库将数据改变记录到二进制日志中，从库读取二进制日志并应用这些改变，以保证从库中的数据与主库中的数据保持一致

​

​

MySQL 主从复制的优点

  • 通过将读操作分配到从库，可以减轻主库的负载，提高系统的性能和可扩展性
  • 通过从库的冗余备份，可以提高数据的可靠性和容错能力
  • 可以使用从库进行数据分析、报表等操作，而不会影响主库的性能
  • 可以实现数据的异地备份，以防止数据灾难

​

​

MySQL 主从复制的缺点

  • 从库的数据可能与主库的数据存在一定的延迟，这可能导致数据不一致
  • 主库的写操作需要同步到从库，这可能会影响主库的性能
  • 如果主库发生故障，需要手动切换到从库，这可能需要一定的时间和人工干预
  • 主从复制的配置和管理比较复杂，需要一定的技术水平和经验

​

​

主从复制可能会出现的问题

主从复制可能会出现的问题：

  • 主数据库故障，导致从数据库无法获取数据变化
  • 从数据库延迟，导致从数据库中的数据不是最新的
  • 从数据库与主数据库的数据不一致，可能是由于数据冲突或复制错误导致的

​

如何解决：

  • 使用主从复制的监控工具来监控主从复制的状态，及时发现并解决问题
  • 使用数据同步工具来解决主从数据不一致的问题
  • 使用延迟复制和半同步复制来解决主从数据库的延迟问题

​

​

如何配置 MySQL 主从复制

  • 修改主库配置文件：配置 server-id，开启二进制日志
  • 在主库上创建一个可以被从库连接的用户，并赋予 REPLICATION SLAVE（复制从节点） 权限。查看二进制日志文件名和位置
  • 修改从库配置文件：配置 server-id，开启二进制日志，开启中继日志，配置只读
  • 从库使用 CHANGE MASTER TO 连接主库，启动复制线程

什么是索引

索引是一种数据结构，就像是一本书的目录，使用索引可以快速定位到需要查询的数据，用于提高数据库查询效率

​

​

为什么要使用索引

优点：

  • 加快数据检索速度，提高数据查询效率
  • 创建唯一性索引，可以保证数据表中每一行数据的唯一性
  • 帮助引擎层避免排序和临时表
  • 将随机 I/O 变为顺序 I/O，加速表和表之间的连接

​

缺点：

  • 占用额外空间，影响插入速度

​

​

索引的类型

MySQL 中有多种类型的索引，包括 B-tree 索引、哈希索引、全文索引等。B-tree 索引是最常用的一种索引类型，它可以用于加速基于范围查询的操作

​

​

聚簇索引和非聚簇索引

​

​

如何优化索引

  • 在经常被查询的列上创建索引，特别是在数据表中的主键和外键上创建索引
  • 根据具体情况创建合适的索引类型，例如 B-tree 索引、哈希索引等。
  • 避免创建过多的索引，因为过多的索引会降低插入、更新和删除操作的性能。
  • 定期进行索引优化和重建操作，以保证索引的性能和稳定性

​

​

主键与外键

  • 主键是一种唯一标识符，用来唯一标识数据库表中的每一行数据
  • 外键是一种限制条件，用于保证数据的完整性和一致性，它指向另一个表的主键，用于建立表与表之间的关系

​

数据库备份

在不影响其他事务的前提下备份和恢复数据库可以使用 XtraBackup 备份工具，XtraBackup 支持不锁表备份数据库，因此可以在备份时继续执行其它事务。同时，它还可以进行增量备份，只备份已更改的数据，可以大大减少备份时间

什么是数据库高可用

MySQL 数据库的高可用是指在 MySQL 数据库中使用多个节点来保证数据库的可用性和容错性，当一个节点故障时，其它节点可以自动接管它的工作，从而保证系统的正常运行

​

​

数据库高可用方案

  • 主从复制：将一个数据库的数据复制到另一个或多个数据库中，其中一个数据库为主数据库，其它数据库为从数据库，从数据库会自动复制主数据库中的数据变化
  • 多主复制：多个数据库之间相互复制数据，每个数据库既是主数据库也是从数据库，可以实现高可用和负载均衡
  • MySQL Cluster：使用 MySQL Cluster 架构来实现数据库的高可用，数据存储在多个节点上，节点之间相互复制，可以实现自动容错和负载均衡

​

​

什么是数据库读写分离

MySQL 数据库的读写分离是指将读取操作和写入操作分别分配给不同的数据库节点来处理，可以提高系统的性能和稳定性

​

​

数据库读写分离原理

MySQL 数据库的读写分离的原理是将读取操作分配给只读从数据库节点，将写入操作分配给主数据库节点，从而减轻主数据库的负载，提高系统的性能和稳定性

​

​

数据库日志

MySQL 提供了多种日志类型，用于记录数据库的各种操作和事件。以下是 MySQL 常见的日志类型：

  1. 错误日志（Error Log）：记录 MySQL 服务器在启动、运行和关闭过程中发生的错误和警告信息。通过查看错误日志，可以了解 MySQL 服务器的健康状态和问题原因
  2. 查询日志（General Query Log）：记录 MySQL 服务器接收到的所有SQL语句，包括查询、更新、删除、插入等操作。通过查看查询日志，可以了解数据库的访问情况和性能瓶颈
  3. 慢查询日志（Slow Query Log）：记录执行时间超过指定阈值的 SQL 语句，用于识别慢查询和优化性能。通过查看慢查询日志，可以了解哪些 SQL 语句需要优化或索引
  4. 二进制日志（Binary Log）：记录所有更改数据库的操作，包括数据的增删改和表结构的修改。二进制日志可以用于数据恢复、主从复制和数据迁移等场景
  5. 事务日志（Transaction Log）：记录数据库中每个事务的操作，包括事务的开始、提交、回滚等。事务日志可以用于数据恢复和崩溃恢复等场景
  6. 中继日志（Relay Log）：用于 MySQL 主从复制场景中，记录从服务器接收到的二进制日志，并将其解析为 SQ L语句。中继日志可以用于保证主从数据一致性和故障恢复

Docker 篇

什么是 Docker

Docker 是一个开源的容器平台，可以帮助开发人员和运维人员更轻松的构建、打包、部署和运行应用程序

​

​

什么是容器

容器是一种沙盒技术，具有轻量级、可移植并且互相隔离的特点；利用容器可以把应用程序运行时所需要的环境依赖封装起来，使应用程序可以在任何地方以相同的方式运行

​

​

Docker 架构组成

  • Docker 客户端：使用 Docker API 与 Docker 守护进程进行通信的命令行工具
  • Docker 守护进程：Docker 守护进程是在主机上运行的服务，负责管理 Docker 容器、镜像、网络和数据卷等。它与 Docker 客户端通过 Docker API 进行通信
  • Docker 镜像：Docker 容器的模板，本质上是一些程序文件的集合
  • Docker 容器：Docker 容器是基于 Docker 镜像创建的可运行实例
  • Docker 仓库：镜像仓库，保存镜像的地方

​

​

Docker 与虚拟机

架构：

  • Docker 是基于容器的虚拟化技术，运行在操作系统之上，共享宿主机的内核，将应用程序及其依赖项打包在一个容器中
  • 虚拟机是基于硬件的虚拟化技术，它模拟出一台完整的计算机，包括操作系统、硬件等

​

资源利用率：

  • 虚拟机需要模拟整个操作系统，需要更多的资源，而 Docker 直接利用宿主机的操作系统内核，资源利用率更高

​

启动速度：

  • Docker 启动快速，属于秒级别；虚拟机通常需要分钟时间启动

​

安全性：

  • 虚拟机可以提供更好的系统隔离，因为每个虚拟机都有自己的操作系统内核，而 Docker 只是利用宿主机的操作系统内核，因此隔离性相对较差

​

​

Docker 核心技术

Docker 容器技术的实现依赖于 Linux 内核中的 Namespace 与 cgroup 技术。其中 Namespace 叫做名称空间，提供容器间的资源隔离功能，cgroup 提供资源限制、分配功能

​

​

Docker 优点

  • 快速部署：短时间内可以快速部署多个容器，实现快速交付
  • 高效虚拟化：基于 Linux 内核实现应用虚拟化，不需要 Hypervisor 的额外支持，相比虚拟机大幅提升性能和效率
  • 节省开支：提高服务器利用率，降低 IT 支出
  • 简化配置：将运行环境打包保存至容器，直接启动即可使用
  • 环境统一：将开发、测试、生产等应用的运行环境进行标准化和统一，减少环境不一致带来的各种问题

​

​

Dockerfile 是什么

Dockerfile 是一种可以被 Docker 程序解释执行的脚本文件，其中包含构建 Docker 镜像所需要的指令。通过 Dockerfile 文件可以直观的描述出 Docker 镜像的构建过程，Docker 程序读取 Dockerfile 文件中的指令并生成 Docker 镜像，可以根据实际业务需要构建符合生产环境的 Docker 镜像，避免手动重复部署镜像的繁琐过程

​

​

Dockerfile 常用指令

指令 介绍
FROM 指定基础镜像
LABEL 指定镜像元数据（比如：作者、版本等）
WORKDIR 设置容器工作目录
RUN 构建镜像阶段执行的 SHELL 命令
ARG 设置构建镜像阶段环境变量
COPY 拷贝文件或者目录到镜像中
ADD 拷贝文件或者目录到镜像中并自动解压缩
EXPOSE 容器对外暴露的服务端口
VOLUME 匿名卷，用于数据保存和持久化工作
USER 指定运行容器阶段的用户；默认为 root
HEALTHCHECK 健康检查：检查容器的健康性
ENV 设置运行容器阶段环境变量
CMD 运行容器阶段执行的 SHELL 命令；只有最后一个生效，CMD 会被 docker run 之后的参数替换
ENTRYPOINT 运行容器阶段执行的 SHELL 命令；只有最后一个生效，一定会执行，不会被忽略，docker run 之后的参数会被追加

​

​

CMD & ENTRYPOINT

在 Dockerfile 中，CMD 与 ENTRYPOINT 指令都可以实现在运行容器阶段时执行 SHELL 指令

  • CMD：只有最后一个生效，CMD 会被 docker run 之后的参数替换
  • ENTRYPOINT：只有最后一个生效，一定会执行，不会被忽略，docker run 之后的参数会被追加 ENTRYPOINT 后面

​

​

Docker 网络通信

在成功部署 Docker 之后，Docker 默认会生成一个 docker0 的虚拟网桥，容器会连接到这个网桥上，并且会自动分配一个IP地址。容器之间可以通过IP地址进行通信，也可以通过容器名称进行访问。因为同一宿主机内的容器都连接同一网桥，所以一容器之间默认互通

​

​

Docker 网络模式

Docker 网络支持 5 种网络模式：

网络模式 描述
Birdge 网桥。Docker 默认使用的网络模式，在宿主机上生成一个虚拟网桥，将容器连接在网桥之上，容器之间通过网桥进行通信
Host 容器不会创建自己的虚拟网卡，而是共享宿主机网络，直接使用宿主机的网卡和 IP 地址
None 关闭容器网络功能
Container 容器连接到另一个容器的网络栈上，与该容器共享一个网络
自定义网络 自定义配置网段地址、网关等信息

Nginx 篇

什么是 Nginx

Nginx 是一款高性能的开源的 Web 服务器和反向代理服务器，具有轻量级、高并发、低内存消耗、可扩展性强的特点

​

​

Nginx 常见模块

Nginx 有很多常见的模块，包括：

  • HttpCoreModule：Nginx 的核心模块，包含了 HTTP 请求处理的基本功能
  • HttpProxyModule：Nginx 的反向代理模块，可以将客户端的请求转发到后端的应用服务器，实现负载均衡和高可用性
  • HttpUpstreamModule：Nginx 的上游模块，可以用于定义后端服务器的集群，负责管理和监控后端服务器的健康状态
  • HttpRewriteModule：Nginx 的 URL 重写模块，可以用于修改客户端请求的 URL，包括重定向、替换等操作
  • HttpSslModule：Nginx 的 SSL 模块，可以用于实现 Web 服务器的 HTTPS 访问，提供 SSL 证书的管理和配置

​

​

Nginx 负载均衡

Nginx 的负载均衡可以通过以下几种方式实现：

  • 基于轮询（Round Robin）的负载均衡：Nginx 将请求均匀地分配到每个后端服务器上
  • 基于 IP 哈希的负载均衡：Nginx 根据客户端 IP 地址的哈希值将请求分配到特定的后端服务器上
  • 基于权重的负载均衡：Nginx 根据后端服务器的权重值分配请求，权重越高的服务器获得的请求越多
  • 基于最小连接数的负载均衡：Nginx 将请求分配到当前连接数最少的后端服务器上

​

​

Nginx 架构

Nginx 使用 EPOLL I/O 多路复用技术，支持高并发。Nginx 是多进程组织模型，由一个 Master 主进程和若干 Worker 工作进程组成

  • Master：读取配置文件和创建工作进程
  • Worker：处理客户端请求

​

​

Nginx 为什么快

Nginx 之所以快，是因为 Nginx 采用了 epoll 事件驱动机制。epoll 是一种基于事件驱动的 I/O 模型，用于监听客户端连接和处理请求，它可以让程序在等待事件时不会被阻塞，而是可以执行其他任务。在 Nginx 中，epoll 模型可以同时监听多个客户端连接和请求，并在事件发生时立即通知应用程序，从而提高服务器的并发处理能力和响应速度

一句话概述：运用了 epoll 模型，提供了一个队列，排队解决

​

​

Nginx 与 Apache 选择

  • Nginx：适用于高并发、高负载的场景，能够实现反向代理与负载均衡，能够静态文件的快速访问
  • Apache：适用于中小型网站，特别是 PHP 网站，对于动态文件的处理能力优于 Nginx

​

​

Nginx 和 Apache 区别

  • 架构设计：Apache 是多线程模型，每个线程负责处理一个请求，而 Nginx 是事件驱动模型，采用异步非阻塞方式处理请求，可以同时处理多个连接
  • 内存占用：Nginx 的内存占用比 Apache 更少，因为 Nginx 的处理方式是事件驱动的，不需要为每个连接创建一个线程或进程
  • 高并发能力：Nginx 在高并发场景下的性能表现更优异，可以支持更多的并发连接
  • 可扩展性：Nginx 的可扩展性比 Apache 更好，可以通过添加模块来扩展其功能，而 Apache 的模块化设计比较复杂，不太容易扩展

​

Kubernetes 篇

什么是 Kubernetes

Kubernetes 是一个开源的容器编排平台，用于管理和部署容器化应用程序。为容器化的应用提供部署运行、资源调度、服务发现和动态伸缩等一系列完整功能，提高了大规模容器集群管理的便捷性

​

​

Kubernetes 组件

Kubernetes 控制节点组件：

  • API Server：声明式 API，整个集群的 API 网关。Kubernetes 系统的入口，为集群中各种资源对象的增删改查等操作提供了接口
  • Kube-Controller-Manager：控制器管理器，集群内部的的管理控制中心。负责执行各种控制器保证 Kubernetes 的正常运行，将 API 声明转化为具体的实现
  • Kube-Scheduler：调度器。为未完成绑定的 Pod 进行节点分配，负责集群的资源调度
  • Etcd：一个高可用的键值对存储系统。保存整个集群的状态，存储集群中所有需要持久化的数据，主要用于配置共享和服务发现

​

Kubernetes 工作节点组件

  • Kubelet：运行在每个 Worker 节点的代理组件，Pod 生命周期管理组件。负责接收并执行控制节点发来的指令，管理由调度器分配到当前节点上的 Pod 对象
  • CRI：对接外部的容器运行时
    • Container RunTime（容器进行时）：负责镜像管理理及 Pod 和容器的真正运⾏，为 Kubernetes 创建的容器提供网络、通信、持久化存储等资源
  • Kube Proxy：运行在每个 Worker 节点之上。Service 功能在节点级的实现，具体来说就是把 Service 的定义转换为节点上的 Iptables 或 Ipvs 规则。核心功能是将到某个 Service 的访问请求转发到后端的多个 Pod 实例上

​

​

K8s 和 Docker 的关系

Docker 提供容器的生命周期和管理，而 Kubernetes 用于关联和编排在多个主机上运行的容器

​

​

什么是 Pod

Pod 是 Kubernetes 中最小调度单元，最小的可部署对象，它是一个包含一个或多个容器的容器组

​

​

什么是 Deployment

Deployment 是 Kubernetes 中一种用于管理 Pod 的对象，它可以确保指定数量的 Pod 始终运行，并提供滚动更新和回滚功能

​

​

什么是 Service

Service 是 Kubernetes 中一种用于暴露 Pod 的对象，它可以提供一个固定的 IP 地址和 DNS 名称，以便其他应用程序可以访问该 Pod

​

​

ipvs 和 iptables

iptables 与 IPVS 都是基于内核实现的，两者的定位不同，iptables 是为防火墙而设计的，IPVS 则专门用于高性能负载均衡

​

​

​

Redis 篇

什么是 Redis

Redis 是一个开源的高性能的键值对数据库，与传统数据库不同的是，Redis 是保存在内存中的，所以读写速度非常快，因此 Redis 被广泛应用于缓存方向，每秒可以处理超过 10 万次读写操作

​

​

缓存数据的处理流程

  1. 如果用户请求的数据在缓存中就直接返回
  2. 缓存中不存在的话就看数据库中是否存在
  3. 数据库中存在的话就返回并更新缓存中的数据
  4. 数据库中不存在的话就返回空数据

​

​

Redis 的线程模型

Redis 6.0 前使用单线程方式接收处理用户请求：Redis 在处理客户端的请求时，包括对数据的读取和操作都由一个顺序串行的主线程处理，避免了多线程切换等性能开销

Redis 6.0 后可开启多线程接收处理用户请求，主要是为了提高网络 I/O 的读写性能，充分利用服务器 CPU 资源
