什么是 Kubernetes

Kubernetes 是一个开源的容器编排平台，用于管理和部署容器化应用程序。为容器化的应用提供部署运行、资源调度、服务发现和动态伸缩等一系列完整功能，提高了大规模容器集群管理的便捷性

​

​

Kubernetes 组件

控制节点是 Kubernetes 集群中的管理节点，负责整个集群的管理和控制：

  • API Server：声明式 API，为 Kubernetes 集群中各种资源对象的增删改查等操作提供了接口，用于接收和处理来自客户端的请求
  • Kube-Controller-Manager：控制器管理器，集群内部的的管理控制中心。负责执行各种控制器保证 Kubernetes 的正常运行，将 API 声明转化为具体的实现
  • Kube-Scheduler：调度器。为未完成绑定的 Pod 进行节点分配，负责集群的资源调度
  • Etcd：一个高可用的键值对存储系统。用于存储 Kubernetes 集群的所有配置信息和状态信息，是 Kubernetes 的后端数据库，主要用于配置共享和服务发现

​

Kubernetes 工作节点组件

  • Kubelet：运行在每个 Worker 节点上，Pod 生命周期管理组件。负责接收并执行控制节点发来的指令，管理由调度器分配到当前节点上的 Pod 对象
  • Container RunTime（容器进行时）：负责镜像管理理及 Pod 和容器的真正运⾏，为 Kubernetes 创建的容器提供网络、通信、持久化存储等资源
  • Kube Proxy：运行在每个 Worker 节点之上。Service 功能在节点级的实现，具体来说就是把 Service 的定义转换为节点上的 Iptables 或 Ipvs 规则。核心功能是将到某个 Service 的访问请求转发到后端的多个 Pod 实例上

​

​

K8s 和 Docker 的关系

Docker 提供容器的生命周期和管理，而 Kubernetes 用于关联和编排在多个主机上运行的容器

​

​

什么是 Pod

Pod 是 Kubernetes 中最小调度单元，是一个或多个容器的集合

​

​

K8s 控制器

  • ReplicaSet（副本集）：用于保证指定数量的 Pod 正常运行，并且在 Pod 出现故障或需要进行扩缩容时，自动创建或删除 Pod 副本
  • Deployment（部署）：用于管理 ReplicaSet 控制器，并且可以实现滚动更新和回滚等功能
  • StatefulSet（有状态副本集）：用于管理有状态的应用程序，保证 Pod 的唯一性、有序性和稳定性
  • DaemonSet（守护进程集）：用于在每个节点上运行一个 Pod 副本，通常用于运行一些守护进程程序
  • Job（作业）：用于运行一次性任务，例如批处理任务、数据处理等
  • CronJob（周期性作业）：用于周期性地运行一次性任务，例如定时备份、定时清理等

​

​

什么是 Service

在 Kubernetes (K8s) 中，Service 是一种抽象，用于定义一组 Pod 的逻辑网络端点，Service 有一个唯一的 IP 地址和端口，这个 IP 地址和端口将作为该 Service 的访问入口，客户端可以使用这个 IP 地址和端口来访问 Service

​

​

Service 类型

  1. ClusterIP：默认的 Service 类型，只能在集群内部使用，为 Service 分配一个虚拟 IP 地址，通过这个 IP 地址可以访问 Service 中的 Pod
  2. NodePort：为 Service 暴露一个固定的端口号，可以通过节点的 IP 地址和这个端口号来访问 Service 中的 Pod。并且可以被外部网络访问
  3. LoadBalancer：通过云供应商的负载均衡器（如 AWS ELB、GCP LoadBalancer 等）来暴露 Service，这样可以将请求负载均衡到多个节点上，提高应用程序的可用性和可扩展性。
  4. ExternalName：将 Service 映射到一个集群外部的 DNS 名称，将集群外部的服务引入到集群内部来，用于访问外部服务，例如数据库、缓存等

​

​

ipvs 和 iptables

iptables 与 IPVS 都是基于内核实现的，两者的定位不同，iptables 是为防火墙而设计的，IPVS 则专门用于高性能负载均衡

​

​

K8s 的探针类型

Kubernetes 有三种探针类型：启动探针、存活探针和就绪探针

  • 启动探针：探测容器启动过程中是否发生问题，只运行在容器初始化期间
  • 存活探针：对容器内的进程进行探测，检测容器是否正常运行
  • 就绪探针：探测容器内服务是否正常工作

​

​

K8s 探测方法

  1. 在容器内执行指定命令，根据命令结果的状态码判定探测结果
  2. 根据相应 TCP 套接字连接建立状态判定。对指定端口上容器的 IP 地址进行 TCP 检查，如果端口打开，则认为诊断成功
  3. 根据指定 HTTP/HTTPS 服务 URL 的响应结果判定。对指定的端口和路径上容器的 IP 地址执行 HTTP Get 请求，如果响应的状态码大于等于 200 且小于 400，则认为是诊断成功

​

​

Pod 状态

在 Kubernetes 中，Pod 可以处于以下几种不同的状态：

  1. Pending（等待中）：Pod 已经被 Kubernetes API 接受，但是还没有被调度到具体的节点上运行
  2. Running（运行中）：Pod 已经被调度到节点上，并且其中至少一个容器正在运行
  3. Succeeded（成功）：Pod 中的所有容器都已经成功地运行并且已经退出
  4. Failed（失败）：Pod 中至少有一个容器已经退出，并且退出状态是非零状态
  5. Unknown（未知）：Pod 的状态无法被确定，通常是由于与 Pod 相关的节点或者控制器出现了故障
