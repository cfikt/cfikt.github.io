Zabbix 是什么

Zabbix 是一个分布式开源的监控解决方案，可以对各种 IT 资源进行监控和管理，包括服务器、网络设备、应用程序等

​

​

Zabbix 架构

Zabbix 的架构包括：

  • Zabbix Server：负责接收和处理来自代理、被监控主机和其他 Zabbix Server 的数据，并将数据存储在数据库中
  • Zabbix Agent：运行在被监控主机上，负责收集主机的数据并将数据发送到 Zabbix Server 或 Zabbix Proxy
  • Zabbix Web 界面：提供了一个用户友好的界面，用于管理和监控被监控主机
  • Database：数据库，存放所有配置信息以及 Zabbix 采集到的数据

​

​

Zabbix工作模式

Zabbix 有两种工作模式：默认为被动模式，生产环境建议使用主动模式

  • 主动模式：Zabbix Server 给 Zabbix Agent 发送一个监控项的任务清单，Zabbix Agent 根据任务清单，采集在内的所有监控项的值，主动将其汇报给 Zabbix Server（将所有监控项的值打包，只需要发送一次数据，大幅度提高了传输效率并且降低服务器压力）
  • 被动模式：Zabbix Server 从 Agent 捕获数据默认使用的是被动模式，即 Zabbix Server 需要往 Agent 上捕获数据，监控项个数有多少，就需要捕获多少次（交互次数取决于监控项个数，若监控项太多，则对 Zabbix Server 造成负载太大）

  • 无论是模式还是被动模式，都是站在 Zabbix Agent 角度来说的工作模式；

  • 两种模式可以在 Zabbix Server 上同时存在，可以将一部分监控项设为主动模式，其它设为被动模式

​

​

监控主机实现流程

  1. 安装 Agent
  2. 配置 Agent 指向 Zabbix Server
  3. Zabbix Web 添加被监控主机
  4. Zabbix Web 在被监控主机上关联模板
  5. 观察结果

​

​

自定义监控项实现流程

  1. 被控端创建自定义监控项（重启服务）
  2. 服务器端使用 zabbix_get 测试
  3. Zabbix Web 创建模板
  4. Zabbix Web 添加自定义监控项到该模板
  5. Zabbix Web 在被监控主机上关联该模板
  6. 观察结果

​

​

自定义模板实现流程

所谓模板，就是一些相关的监控项的集合，还可以在模板中包括图形、触发器等；

  1. 创建模板，模板必须属于某个主机组（一般属于主机组 Templates）
  2. 在模板中创建监控项、图形、触发器
  3. 创建需要监控的主机，然后关联对应的模板
  4. 更改模板的监控项目，所有使用模板的主机会自动更改
  5. 导出模板，后期可以至其他系统继续使用

​

​

Zabbix 可以监控哪些指标

Zabbix 可以监控的指标有很多，包括服务器的 CPU 、内存使、磁盘等系统资源使用情况；网络设备的带宽、流量、丢包率等网络性能指标；应用程序的响应时间、并发连接数、错误率等应用性能指标

​

Zabbix 的告警方式

Zabbix 提供了多种告警方式，包括邮件、短信、微信等
