什么是 Keepalived

Keepalived 是一款开源的高可用性解决方案，基于 VRRP 协议实现，可以用于实现集群的 IP 虚拟化、服务的故障切换等。Keepalived 通过检测服务器的可用性来确保 VIP 是始终可用的，并且可以自动切换到备份节点实现 VIP 的动态漂移以确保服务的连续性

​

​

​

​

Keepalived 的作用与原理

Keepalived 是一个基于 VRRP 协议的高可用性解决方案。它可以将多个服务器组成一个虚拟 IP 地址，实现对服务的高可用性保障。其原理是通过 VRRP 协议，将多个服务器组成一个虚拟路由器，对外提供一个虚拟 IP 地址，当主服务器出现故障时，备用服务器可以接管服务，保证服务的持续可用性

​

Keepalived 工作原理基于 VRRP（Virtual Router Redundancy Protocol）协议。在 Keepalived 中，虚拟路由器组（VRG）是由一个 Master 节点和多个Backup 节点组成的

  • Master 节点具有最高的优先级，负责将 VIP 绑定到自己的网络接口上，并将 VRRP 广播到网络中
  • Backup 节点只是监听 Master 节点的状态，当 Master 节点失效时，其会发现并将自己切换为 Master 节点

​

​

​

​

VRRP 概述

VRRP（Virtual Router Redundancy Protocol）是一种网络协议，可以将多台路由器组成一个虚拟路由器组（VRG），并将虚拟路由器组看作是一台物理路由器，对外提供虚拟 IP 地址和虚拟 MAC 地址，从而实现路由器的冗余和故障转移

​

​

​

​

Keepalived 的健康检查方式有哪些？

  • TCP 检查：检查服务是否能够建立 TCP 连接。
  • HTTP 检查：发送 HTTP 请求并检查响应状态码。
  • SMTP 检查：检查 SMTP 服务是否正常运行。
  • LVS 检查：检查 Linux Virtual Server 是否正常工作。
  • 脚本检查：运行自定义脚本进行检查

​

​

​

​

Keepalived 的负载均衡算法有哪些？

  • 轮询算法（Round Robin）：依次将请求转发到每个服务器。
  • 加权轮询算法（Weighted Round Robin）：根据服务器的权重将请求转发到不同的服务器。
  • 最小连接数算法（Least Connections）：将请求转发到连接数最少的服务器。
  • 加权最小连接数算法（Weighted Least Connections）：根据服务器的权重将请求转发到连接数最少的服务器。
  • IP 哈希算法（IP Hash）：根据客户端的 IP 地址将请求转发到固定的服务器
