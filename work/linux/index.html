什么是进程

进程是正在运行程序的实例，是操作系统中的一个基本概念，每个进程都有自己的内存空间、程序代码和数据。操作系统通过管理进程来协调计算机资源的分配和使用

​

​

​

​

进程的基本状态

进程的基本状态包括：

  1. 新建状态（Created）：进程被创建但还没有开始执行
  2. 就绪状态（Ready）：进程已经准备好执行，但还没有被分配到 CPU 资源
  3. 运行状态（Running）：进程正在执行，占用 CPU 资源
  4. 阻塞状态（Blocked）：进程因为某种原因暂时无法执行，等待外部事件的发生，如等待输入/输出、等待资源等
  5. 终止状态（Terminated）：进程执行完毕或被操作系统强制终止，进程的资源被彻底释放

进程和线程

  • 进程：进程是正在运行程序的实例，是操作系统中的一个基本概念，每个进程都有自己的内存空间、程序代码和数据。操作系统通过管理进程来协调计算机资源的分配和使用

  • 线程：进程划分成的更小的运行单位，是操作系统能够进行运算调度的最小单位

进程和线程都是系统中的执行单位，但是它们的调度、资源占用、通信和安全性等方面有所不同

Linux 性能优化

对于 Linux 操作系统的性能优化操作有很多，最直观的就是硬件优化了，在系统上配置更快内存更大的磁盘以及更高速率的网卡或者增加更多的 CPU 和内存都可以提高操作系统的性能

​

最节约成本也是我最常用的办法就是对操作系统的内核参数进行调优，对于 Linux 操作系统，可以使用 sysctl -a 查看内核参数的配置，在 /etc/sysctl.conf 文件直接编写就可以对内核参数进行修改。较常见的有

  1. 开启 IPv4 转发
  2. 开启 TCP 时间戳
  3. 允许非本地 IP 地址 Socket 监听
  4. 开启 TCP 复用
  5. 开启 TCP 连接的快速回收
  6. 调整 TCP 连接的超时时间
  7. 开启 SYN Cookies 防止 DDoS 攻击
  8. 修改 SWAP 的占比
  9. …

还可以修改 limits.conf 文件，对系统资源限制进行配置，对 “打开文件的最大数目”，“开启进程/线程的最大数目”等进行优化

​

还可以使用 Linux 性能分析工具对操作系统进行分析，来找出性能瓶颈

Linux 内核优化参数

文件系统：

  • vm.dirty_background_ratio：指定当脏数据占内存的百分比时，后台进程开始将脏数据写回磁盘。默认值为 10%
  • vm.dirty_ratio：指定当脏数据占内存的百分比时，系统将停止所有写入操作，直到脏数据被写回磁盘。默认值为 20%
  • vm.swappiness：指定内存不足时，系统将会使用交换分区的程度。默认值为 60
  • fs.file-max：指定系统可以打开的文件描述符的最大数量。默认值为 10 万

​

网络协议栈：

  • net.core.somaxconn：指定监听队列的最大长度。默认值为 128
  • net.core.netdev_max_backlog：指定网络接口队列的最大长度。默认值为 1000
  • net.ipv4.tcp_fin_timeout：指定 TCP 连接终止时等待 FIN 包的时间。默认值为 60 秒
  • net.ipv4.tcp_tw_reuse：开启 TCP 连接复用，即允许新的 TCP 连接使用之前的已经处于 TIME_WAIT 状态的连接。默认值为 0
  • net.ipv4.tcp_tw_recycle：开启 TCP 连接快速回收，即将处于 TIME_WAIT 状态的连接快速回收以节省系统资源。默认值为 0

​

内存管理：

  • vm.min_free_kbytes：指定系统保持的空闲内存的最小值。默认值为 204 8KB
  • vm.max_map_count：指定一个进程可以拥有的内存映射数量。默认值为 65536

简述 Linux 系统性能优化

Linux 系统性能优化是指通过调整系统的各种参数和优化系统的各个组件，使系统能够更好地满足用户的需求，提高系统的性能和稳定性

​

​

​

​

Linux 系统性能分析查看

  1. 可以使用命令 uptime 来查看系统的负载情况，如果系统的负载值超过了 CPU 的核心数，则说明系统存在性能问题
  2. 可以使用命令 top 来查看 Linux 系统的 CPU 使用情况，如果发现某个进程占用了大量的 CPU 资源，则可以通过优化该进程或者使用 CPU 限制工具来优化 CPU 性能
  3. 可以使用命令 free 来查看 Linux 系统的内存使用情况，如果发现系统的内存使用率过高，则可以通过优化进程或者调整内存参数来优化内存性能
  4. 可以使用命令 df 来查看 Linux 系统的磁盘使用情况，如果发现磁盘空间不足或者磁盘读写速度过慢，则可以通过清理磁盘空间或者升级磁盘硬件来优化磁盘性能
  5. 可以使用命令 iftop 来查看 Linux 系统的网络使用情况，如果发现网络带宽不足或者网络延迟过高，则可以通过优化网络设备或者调整网络参数来优化网络性能
  6. 可以使用命令 ps 来查看 Linux 系统的进程使用情况，如果发现某个进程占用了大量的系统资源，则可以通过优化该进程或者使用进程限制工具来优化进程性能
  7. 可以使用命令 dmesg 和 /var/log/syslog 来查看系统日志，通过分析日志信息可以发现系统的性能问题，进而采取相应的措施来解决问题

Linux 性能工具

常用的性能工具包括：sar、top、vmstat、iostat、netstat 等，这些工具可以帮助我们更好地了解系统的各个方面的性能情况，从而采取相应的措施来优化系统性能

  1. sar：可以收集系统的各种性能数据，如 CPU 使用率、内存使用率、磁盘使用率等，并将这些数据存储在文件中，方便后续分析
  2. top：可以实时监控系统的进程情况，包括进程的 CPU 使用率、内存使用率等，可以快速定位系统性能问题
  3. vmstat：可以监控系统的虚拟内存使用情况，包括内存使用率、交换分区使用率等，可以帮助我们了解系统的内存使用情况
  4. iostat：可以监控系统的磁盘 I/O 使用情况，包括磁盘读写速度、磁盘 I/O 等待时间等，可以帮助我们了解系统的磁盘性能情况
  5. netstat：可以监控系统的网络使用情况，包括网络连接数、网络带宽使用情况等，可以帮助我们了解系统的网络性能情况
