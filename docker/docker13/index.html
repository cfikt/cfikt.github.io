Docker Compose 简介

当在一个宿主机内启动较多容器时，如果手动操作会比较繁琐且容器出错，也比较难以控制多个容器之间的启动顺序或者依赖关系，此时推荐使用 Docker 官方的单机编排工具 docker-compose

​

docker-compose 是 Docker 官方的一种单机编排服务，是一个管理多个容器的工具。使用 docker-compose 可以解决容器之间的依赖关系，并且可以替代 docker 命令对容器创建、启动和停止等手工操作

​

​

Docker Compose 安装

docker-compose 安装拥有多种方法：

  1. 通过 Python 安装工具 pip 在线安装
  2. 直接通过 yum 仓库在线安装
  3. 提前将安装包下载到本地，使用安装包离线安装

​

pip 在线安装

pip 是 Python 的一个安装工具，类似于 CentOS 的 yum 以及 Ubuntu 的 apt 命令。通过 pip 可以安装最新版本的 docker-compose，推荐使用

​

yum 在线安装

直接通过系统包仓库使用 yum 在线安装版本较旧，不建议使用

​

离线安装

提前将安装包下载到本地，使用安装包离线安装，适用于无网络环境下安装，此方法可指定安装版本，推荐使用

​

​

docker-compose.yml

Docker Compose 文件是一个 YAML 格式的文件，需要严格注意首行缩进。docker-compose 命令默认会调用当前目录下的 docker-compose.yml 文件，因此执行 docker-compose 命令之前需要先进入 docker-compose.yml 文件所在目录

​

Docker Compose 的 YAML 文件包含 4 个一级标签：

  • version：docker-compose.yml 文件版本，版本不同，语法和格式有所不同。必须指定，并且必须位于文件首行
  • services：定义不同的应用服务
  • networks：用于指引 Docker 创建新的网络
  • volumes：用于指引 Docker 来创建新的卷
