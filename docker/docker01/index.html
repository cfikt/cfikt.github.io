为什么要学习容器

容器技术是虚拟化、云计算、大数据之后的一门重要的技术，容器技术提高了硬件资源的利用率、方便了业务的互相扩容、实现了宕机自愈等功能；而 Docker 和 Kubernetes 是最典型的代表，在容器技术愈发流行的时代，熟练掌握好该技术的重要性是毋容置疑的

​

​

什么是容器

容器是一种沙盒技术，具有轻量级、可移植并且互相隔离的特点；利用容器可以把应用程序运行时所需要的环境依赖封装起来，使应用程序可以在任何地方以相同的方式运行

​

​

什么是 Docker

注意：Docker 属于容器技术，而容器不仅仅是 Docker，Docker 是实现容器的一种代表技术

Docker 是一个开源的应用容器引擎，是一个应用打包、分发、部署的工具，Docker 让开发者可以将应用程序以及环境打包到一个可移植的容器中，并移植或发布到任何流行的 Linux 机器上

​

​

Docker & 虚拟机

Docker & 虚拟机

技术比较

  • 虚拟机：虚拟出一个主机硬件，并且运行一个完整的操作系统
  • Docker：直接运行在宿主机的操作系统之上，容器没有自己的内核，多个容器共享主机内核，容器内的应用程序直接运行在宿主机的内核之上；也不需要虚拟硬件

表现比较

  • 启动速度：Docker 启动快速，属于秒级别；虚拟机通常需要分钟时间启动
  • 性能损耗：Dcoker 性能损耗更小，Docker 在操作系统之上进行虚拟化，与内核进行交互，性能优于通过 Hypervisor 层的虚拟化
  • 系统利用率：Docker 更轻量，Docker 架构可以共用一个内核与共享应用程序库，并且只运行所需要的环境依赖，而虚拟机运行一个完整的操作系统；同样的硬件条件，Docker 运行的容器数比虚拟机数量更多，对系统的利用率更高
  • 隔离性：Docker 属于进程之间的隔离，虚拟机可以实现系统级别隔离，与虚拟机相比，Docker 隔离性较弱
