docker ps 列出当前容器
docker top 查看容器内进程

​

启动容器

容器是镜像运行时的实例，可以将容器看作一个应用程序及其依赖环境打包而成的集装箱，其本质就是将镜像中的程序启动后生成的进程。容器与镜像的关系类似于虚拟机与虚拟机模板

​

启动容器用法

使用 docker run 可以启动或者创建一个容器

  • docker run 本质上等于 docker pull + docker start
  • 容器启动后，如果容器内没有前台运行的进程，容器将自动退出并停止

守护式容器

容器启动后，如果容器内没有前台运行的进程，容器将自动退出并停止，在 Docker 中，通常启动的使守护式容器：

  • 能够长期运行
  • 无需交互式会话
  • 适合运行应用程序和服务

进入正在运行的容器

Docker 进入正在运行的容器有两种方法：

  • attach：进入容器内部，对容器的操作会在同一个容器的多个会话界面同步显示，且使用 exit 退出后容器自动关闭
  • exec：在运行中的容器启动新进程，对容器执行单个命令或者进入容器内部，使用 exit 退出后容器仍在运行，推荐使用

​

端口映射

容器启动后，默认情况下，外部网络无法访问容器中的内部网络服务，可以使用端口映射来解决该问题：

  • -P（大写）：将容器事先预定义的所有端口映射为宿主机的随机端口，当停止容器后再启动可能会导致端口发生变化
  • -p（小写）：将容器的预定义的指定端口映射到宿主机的指定端口

​

设置容器端口映射后，可以使用 docker port 命令查看容器的端口映射关系

​

传递运行命令

容器内部需要有一个前台运行的进程才能保持容器的运行，可以在启动容器时候通过传递运行参数实现

​

容器里的 PID 为 1 的守护进程的实现方式：

  • 服务类：启动 Nginx、Tomcat、Apache 等服务，保持运行
  • 命令类：通过 tail -f xxx 等命令实现

​

传递环境变量

使用 docker 命令中的 -e 或 —env-file 选项可以实现在启动容器时直接传递变量到容器内部

其它操作

容器 hosts 文件

通常情况下，在启动容器时，Docker 会自动在容器内部的 hosts 文件中加入容器自己的 ID，并将其解析为容器的 IP，可以使用 —add-host 参数修改容器的 hosts 文件

​

指定容器 DNS

通常情况下，在启动容器时，默认采用宿主机的 DNS 作为容器的 DNS，可以使用以下方法指定其它 DNS 地址：

  • 修改宿主机的 DNS 地址
  • 在启动容器时使用 —dns=x.x.x.x 参数
  • 在 /etc/docker/daemon.json 文件指定
