Docker 默认网络通信

Docker 服务安装后，默认会在宿主机上生成一个名称为 docker0，IP 地址为 172.17.0.1/16 的虚拟网桥。

​

Docker 启动一个容器时会根据 Docker 网桥的网段分配给容器一个 IP 地址，称为 Container-IP，同时每个容器的默认网关都为 Docker 网桥地址，因为在同一宿主机内的容器都接入同一个网桥，所有容器内部默认互通

​

默认每次新建容器后：

  • 会在宿主机和容器内部各自生成一个网卡，两个网卡成对出现
  • 容器会自动获取一个 172.17.0.0/16 网段的随机地址。默认从 172.17.0.2 开始，第二次容器为 172.17.0.3 …… 以此类推
  • 容器获取的地址并不固定，每次重启容器，都可能导致 IP 地址的变化

容器名称互联

默认情况下，相同宿主机中容器之间默认互通，在新建容器时，Docker 会自动分配容器名称、容器 ID 以及 IP 地址，每次重启容器时，都可能导致容器的 IP 地址发生变化，所以使用容器 IP 地址进行通信的方式及其不稳定。若想实现相对稳定的容器通信，可以使用容器名称互联方案，即给容器起一个固定的名称，通过固定名称实现容器之间的通信

​

容器名称互联实现

给容器起一个固定的名称，docker run 创建容器时，可以使用 --link 选项实现容器名称的引用，其本质就是在容器内部的 /etc/hosts 文件中添加一条容器 IP 与主机名的对应关系，从而实现名称解析

容器别名实现

自定义的容器名称可能后期会发生变化，那么一旦名称发生变化，再使用之前的名称肯定是无法成功调用，每次都进行更改的话又比较麻烦，因此可以使用自定义别名的方式解决，即容器名称可以随意更改，只要不更改别名即可

不同网络容器通信

在 Docker 中，在同一个宿主机内，若使用不同的 Docker 网络创建的容器会因为 Iptables 规则导致默认无法通信

​

若想实现相同宿主机中不同网络之间容器的通信，不建议直接修改 Iptables 规则，可以通过 docker network connect 实现。该方法本质上是将容器内部添加一个希望通信网段的 IP 地址实现通信，并且该方法是单向的，若想要实现互相通信，在两个容器之间都需要操作

​

跨宿主机容器的通信

不同宿主机之间创建的容器默认不能通信，在实际应用场景中，为了保证业务的高可用性，容器通常是跨主机部署的，并且部署在不同宿主机上的容器会进行大量的网络通信，则需要解决不同宿主机之间容器的通信问题

​

跨主机互联实现说明

  1. 跨主机互联指的是不同宿主机之间创建的容器可以互相访问，实现容器跨主机互联的前提是保证宿主机之间网络可以互相通信
  2. Docker 的默认网段都是 172.17.0.x/16，为避免容器 IP 地址的冲突问题，因此建议修改其中一台宿主机的网段
