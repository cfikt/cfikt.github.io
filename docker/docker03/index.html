什么是 Docker

Docker 是一个开源的应用容器引擎，是一个应用打包、分发、部署的工具，Docker 让开发者可以将应用程序以及环境打包到一个可移植的容器中，并移植或发布到任何流行的 Linux 机器上

​

Docker 的主要目标：通过对应用的组件就行 build（构建）、ship（分发）、run（运行），实现“一次封装、到处运行”

​

​

Docker 组成

  • Docker 主机（Host）：也称为宿主机，一个物理机或虚拟机，用于运行 Docker 服务进程与容器
  • Docker 服务端（Server）：Docker 守护进程，用于运行 Docker 容器
  • Docker 客户端（Client）：客户端使用 docker 命令或其他工具调用 Docker API
  • Docker 镜像（Images）：Docker 容器的模板，本质上是一些程序文件的集合
  • Docker 仓库（Registry）：保存镜像的地方，即镜像仓库
  • Docker 容器（Container）：使用镜像生成对外提供一个或一组服务的实例，其本质就是将镜像中的程序启动后生成的进程

Docker 优点

  • 快速部署：短时间内可以快速部署多个容器，实现快速交付
  • 高效虚拟化：基于 Linux 内核实现应用虚拟化，不需要 Hypervisor 的额外支持，相比虚拟机大幅提升性能和效率
  • 节省开支：提高服务器利用率，降低 IT 支出
  • 简化配置：将运行环境打包保存至容器，直接启动即可使用
  • 环境统一：将开发、测试、生产等应用的运行环境进行标准化和统一，减少环境不一致带来的各种问题
  • 快速迁移和扩展：可实现跨平台运行在物理机、虚拟机、公有云等环境，良好的兼容性实现在任意环境运行容器
  • 更好的实现面向服务的架构：通常一个容器只部署运行一个服务，实现分布式应用模型，更加方便的进行横向扩展，减少不同服务之间的相互影响

​

​

Docker 缺点

  • 隔离性：多个容器共用宿主机的内核，各应用之间的隔离不如虚拟机彻底
  • 安全性：Docker 中普通用户具备对容器进行操作的权限
  • 如果容器内进程需要查看和调试，在每个容器内都需要安装相应的工具，造成存储空间的重复浪费
