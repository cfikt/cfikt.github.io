Harbor 高可用

​

Harbor 支持基于策略的 Docker 镜像复制功能，可以实现不同的数据中心、不同的运行环境之间同步镜像，并提供友好的管理界面，大大简化了实际运维中的镜像管理工作。除了传统的基于共享存储方式，跟推荐使用 Harbor 提供的基于镜像复制的方式，配置简单，并且能够实现双向复制功能

​

​

高可用实现

环境准备：

​

安装 Harbor

实现基于镜像复制的方式部署 Harbor 高可用环境，首先需要安装至少两台 Harbor 主机

​

配置 Harbor01

浏览器访问 Harbor01 地址，并登录系统

​

新建项目：

  1. 项目名称：用户自定义
  2. 访问级别：① 公开：上传需要密码，下载不需要密码；② 不公开：上传下载都需要密码。这里选择公开
  3. 存储容量：-1 为不限制

​

新建目标：

系统管理 → 仓库管理 → 新建目标

  1. 提供者：Docker 镜像仓库提供商
  2. 目标名：即 Harbor 项目名称；两个 Harbor 项目名称需配置一致
  3. 目标 URL：需要与其进行同步的 Harbor 地址
  4. 访问 ID：需要与其进行同步的 Harbor 用户名
  5. 访问密码：需要与其进行同步的 Harbor 用户密码
  6. 验证远程证书：使用权威机构颁发的 SSL 证书才需要勾选此项

​

新建规则：

  1. 名称：用户自定义
  2. 复制模式：① Push：把资源从本地仓库推送到远程仓库； ② Pull：把资源从远程仓库拉取到本地仓库
  3. 目标仓库：需要进行复制的仓库
  4. 触发模式：选择事件驱动
  5. 带宽：-1 表示不限制

​

配置 Harbor02

重复 “配置 Harbor01” 操作：浏览器访问 Harbor02 地址，并登录系统

  1. 新建项目：新建与 Harbor 同名项目
  2. 新建目标：目标 URL 修改为 Harbor01 地址
  3. 新建规则：目标仓库选择 Harbor01
