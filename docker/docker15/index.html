Docker 仓库管理

Docker 仓库是用来存放镜像的地方，又称为镜像仓库，将镜像 Push 到仓库中集中保存，可以更方便的管理和使用 Docker 镜像，在需要镜像的时候，只需要从仓库中 Pull 到本地即可

​

Docker 仓库分为公有云仓库和私有云仓库：

  • 公有云仓库：由互联网公司对外公开的仓库。比如 Docker 官方仓库、阿里云仓库等
  • 私有云仓库：私人或组织内部搭建的仓库，一般只为组织内部使用。比如 Docker Registory、Docker Harbor 等

​

​

Harbor 仓库

Harbor 是一个用于存储和分发 Docker 镜像的企业级 Registry 服务器，具有安全、镜像同步复制、漏洞扫描和权限管理等特性。作为一个企业级的 Registry 服务器，Harbor 提供了更好的性能和安全，能够提升用户使用管理和分发镜像的效率，可以用来构建企业内部的 Docker 镜像仓库，是 Docker 私有仓库的典型代表

​

Harbor 功能

  • 基于角色的访问控制：用户与 Docker 镜像仓库通过“项目”进行组织管理，一个用户可以对多个镜像仓库在同一命名空间（Project）里有不同的权限
  • 镜像同步复制：镜像可以在多个 Registry 中同步复制
  • 图形化用户界面：用户可以通过浏览器来浏览，检索当前 Docker 镜像仓库，管理项目和命名空间
  • AD/LDAP 支持：Harbor 可以集成企业内部已有的 AD/LDAP，用于鉴权认证管理
  • 审计管理：所有针对镜像仓库的操作都可以被记录追溯
  • 国际化：已拥有英文、中文、德文、日文和俄文等本地化版本
  • RESTful API：RESTful API 提供给管理员对于 Harbor 更多的操控, 使得与其它管理软件集成变得更容易
  • 部署简单：提供在线和离线两种安装工具， 也可以安装到 vSphere 平台虚拟设备

​

如上图所述，Harbor 由 6 个大模块组成：

  • Proxy：对应一个 Nginx 反向代理组件。将 Notary Client（镜像认证）、Docker Client（镜像上传下载等）和浏览器的访问请求（Core Service）转发到后端服务上

  • UI（Core Service）：对应 Harbor-UI 组件。主要提供四个子功能。

    • ① UI：一个 Web 管理页面 UI
    • ② API：Harbor 暴露的 API 服务
    • ③ Auth：用户认证服务
    • ④ Token（上图中未体现）：负责根据用户在每个 Project 中的 Role 来为每一个 docker push/pull 命令发布一个 Token

  • Registry：对应 Registry 组件。负责存储镜像文件和处理镜像的 pull/push 命令

  • Admin Service：对应 Harbor-Adminserver 组件。系统的配置管理中心

  • Job Sevice：对应 Harbor-Jobservice 组件。负责镜像复制工作，与 Registry 进行通信，从一个 Registry 拉取镜像并推送到到另一个 Registry，并记录 Job_Log

  • Log Collector：对应 Harbor-Log 组件。日志汇总组件

  • DB：对应 Harbor-DB 组件。数据库

新建项目

浏览器访问 Harbor，新建项目：

  1. 项目名称：用户自定义
  2. 访问级别：① 公开：上传需要密码，下载不需要密码；② 不公开：上传下载都需要密码
  3. 存储容量：-1 为不限制
