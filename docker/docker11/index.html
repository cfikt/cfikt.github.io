Docker 网络连接模式

Docker 网络支持 5 种网络模式：

网络模式 描述
Birdge 网桥，默认模式。创建的容器会为每一个容器分配自己的网络 IP 等相关信息，并将容器连接到一个虚拟网桥实现与外界通信
Host 新创建的容器不会创建自己的虚拟网卡，而是共享宿主机网络，直接使用宿主机的网卡和 IP 地址
None 关闭容器网络功能
Container 创建的容器需要指定一个已经存在的容器，与该容器共享一个网络
自定义网络 自定义配置网段地址、网关等信息

​

安装 Docker 后，会自动创建三个网络，分别为 bridge、host、none。默认使用 birdge 模式

​

默认新建的容器使用 Bridge 模式，创建容器时，可以使用 docker run 命令的 —network 选项指定网络模式

​

​

Bridge 网络模式

Bridge 即网桥，是 Docker 的默认网络模式，此模式创建的容器会为每一个容器分配自己的网络 IP 等相关信息，并将容器连接到一个虚拟网桥实现与外界通信。Bridge 可以和外部网络进行通信，通过 SNAT 访问外网，使用 DNAT 可以让容器被外部主机访问，所以此模式也称为 NAT 模式，使用该模式需要宿主机开启路由转发功能（安装 Docker 自动开启）

​

Bridge 网络模式特点：

  • 网络资源隔离：不同宿主机的容器无法直接通信，各自使用独立网络
  • 无需手动配置：容器默认自动获取 172.17.0.0/16 的 IP 地址
  • 可访问外网：利用宿主机的物理网卡，通过 SNAT 连接外网
  • 外部主机无法直接访问容器：可以通过配置 DNAT（即 docker run 命令的 -p 或 -P 选项） 接受外网的访问
  • 性能较低：通过 NAT，网络转换导致性能的损耗
  • 端口管理繁琐：每个容器必须手动指定唯一的端口，容器产生端口冲容

Host 网络模式

使用 Host 网络模式时，新创建的容器不会创建自己的虚拟网卡，而是共享宿主机网络，直接使用宿主机的网卡和 IP 地址，因此在容器内部查看的 IP 信息就是宿主机的信息。访问容器时直接使用宿主机 IP + 容器端口即可

该模式只共享宿主机网络资源，其它资源，如：文件系统、系统进程等仍然与宿主机保持资源隔离

​

Host 网络模式直接使用宿主机网络而无需转换，网络性能基本无损耗，但是各容器内部不能使用同一端口，适用于容器内部端口相对固定的业务

​

Host 网络模式特点：

  • 共享宿主机网络
  • 网络性能无损耗
  • 网络故障排除相对简单
  • 各容器网络无隔离
  • 网络资源无法分别统计
  • 端口管理困难：容易产生端口冲突
  • 不支持端口映射

​

Host 网络模式实现

在创建容器时，使用参数 --network host 指定该容器使用 Host 网络模式

​

None 网络模式

使用 None 网络模式时，Docker 容器将不会进行任何网络配置（没有网卡、IP 与路由等），因此默认无法与外界通信，需要手动添加网卡并且配置 IP 地址信息，极少使用

​

None 网络模式特点：

  • 默认无网络功能，无法和外部通信
  • 无法实现端口映射
  • 适用于测试环境

​

None 网络模式实现

在创建容器时，使用参数 --network none 指定该容器使用 None 网络模式

​

Container 网络模式

使用 Container 网络模式时，创建的容器需要指定一个已经存在的容器，与该容器共享一个网络，除网络外，其它资源仍然保持相互隔离。因此，需要提前规划好两个容器的端口，避免端口冲突，而两个容器之间可以通过 lo 网卡进行通信

​

Container 网络模式特点：

  • 与宿主机网络空间隔离
  • 容器间共享网络空间
  • 适合频繁的容器间的网络通信
  • 直接使用对方的网络，较少使用

​

Container 网络模式实现

在创建容器时，使用参数 --network container:<容器> 指定该容器使用 Container 网络模式

​

自定义网络模式

除了以上的网络模式，Docker 还支持自定义网络模式，可以自定义配置网段地址、网关等信息

自定义网络内的容器可以直接通过容器名进行相互的访问，而无需使用 --link 选项

​

可以使用自定义网络模式，实现不同集群应用的独立网络管理，互不影响，而且在同一个网络内，可以直接利用容器名相互访问，方便快捷
