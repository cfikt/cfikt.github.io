Docker 安装准备

系统版本选择

Docker 目前已经支持多种操作系统的安装运行，比如 CentOS、Ubuntu、RedHat、Debian 等 Linux 操作系统，甚至还支持 Windows 和 Mac 版本，生产环境中通常在 Linux 系统上部署，在 Linux 系统上需要内核版本 3.10 或以上

​

Docker 版本选择

Docker 现今版本分为社区版 Docker CE 以及企业收费版 Docker EE，两者都是每一个季度发布一个新版本，其版本号规则为 YY.MM，例如 17.09 即代表 2017 年 9 月发布

​

Docker 版本建议选择每一个季度发布的版本，即 xx.03、xx.06、xx.09、xx.12 版本，并且建议选择 Kubernetes 官方测试通过的的版本，避免出现不兼容等未知的及不可预估的问题发生

​

Docker 安装方法

Docker 通常使用两种方法进行安装部署：

  • RPM 包安装
  • 二进制包安装

两种安装方法在 Docker 使用上没有区别，RPM 包安装需要依赖于网络环境下才能进行，生产环境一般使用二进制包安装

​

​

RPM 安装

使用 RPM 包安装 Docker 的方法可以参考清华站或者阿里云等镜像站文档，复制粘贴一顿操作即可完成：

  • 清华镜像站安装文档：https://mirrors.tuna.tsinghua.edu.cn/help/docker-ce/

  • 阿里云镜像站安装文档：https://developer.aliyun.com/mirror/docker-ce

Linux 二进制安装

二进制安装适用于无法上网或无法通过包安装方式的情况
