SQL 语句

SQL 语句说明

  • DDL（数据定义语言）：用于在数据库中创建、删除或修改数据库对象等操作；由 CREATE DROP ALTER 指令组成
  • DML（数据操纵语言）：用于对数据库对象中的数据进行操作；主要由 INSTRT UPDATE DELETE 指令组成
  • DCL（数据控制语言）：用于对数据访问权进行控制，设置或更改数据库用户或角色权限；主要由 GRANT 和 REVOKE 指令构成
  • DQL（数据查询语言）：用于对数据库中数据的查询，不会对数据本身进行修改；主要由 SELECT 指令组成

​

数据查询顺序

在 MySQL 中，每执行一个关键字，数据库都会生成对应的一个虚拟表。随着关键字的逐个执行，虚拟表中的数据不断过滤，数据量原来越少，数据也趋近于正确的结果集，直至 SQL 语句执行完毕，返回最终结果

主主复制

MySQL 主主复制：两个节点都可以更新数据，并且互为主从

​

实现主从复制配置项：

  1. 各节点使用一个惟一 server_id

  2. 都启动 binary log（二进制日志） 和 relay log（中继日志）

  3. 创建拥有复制权限的用户账号

  4. 定义自动增长id字段的数值范围各为奇偶

  5. 均把对方指定为主节点，并启动复制线程

半同步复制

默认情况下，MySQL 的复制功能是异步的：主库把 binlog 日志发送给从库即结束，并不验证从库是否接收完毕，容易造成主服务器和从服务器的数据不一致，甚至在恢复时造成数据的丢失

​

半同步复制：为了保证主从数据的一致性问题；Master 处理事务过程中，提交完事务后，必须等至少一个 Slave 将收到的 binlog（二进制日志）写入 relay log（中继日志）返回 ack（确认字符）才能继续执行处理用户的事务
