七层 IP 透传

在 HAProxy 中，四层 IP 透传的限制过多，并且配置复杂，生产环境不建议使用。这里不讨论四层透传

​

当客户端通过代理服务器访问后端服务器时，后端服务器默认看不到客户端 IP，不便于分析和维护。若要实现客户端 IP 对后端服务器透明，可以使用添加请求报文头部信息的方法，实现 IP 透传

压缩功能

对响应给客户端的报文进行压缩，以节省网络带宽，但是会占用部分 CPU 性能，建议在后端服务器开启压缩功能，而非在 HAProxy 上开启压缩

​

​

配置选项

使用 compression algo <algorithm> ... 启用 HTTP 协议中的压缩机制，并指定压缩方式

​

使用 compression type 指定压缩文件类型

​

​

配置范例

后端服务器健康检测

健康检测方法

HAProxy 支持三种后端服务器检测检测的方法：

  1. 基于四层的传输端口做健康检测；只关注后端服务器的端口是否打开，不关注 Web 服务是否能够正常访问。默认健康检测方法
  2. 基于指定 URI 做状态监测，需要访问整个页面资源，占用更多带宽
  3. 基于指定 URI 的头部内容做状态监测，占用较少带宽。建议使用此方式

​

​

七层健康检测

基于应用层 HTTP 协议，采用不同的监测方式，对后端服务器进行状态监测

注意: 此方式会导致后端服务器上生成大量的由 HAProxy 发起的访问日志

​
