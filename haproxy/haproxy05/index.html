ACL 概述

访问控制列表（ACL，Access Control Lists）是一种基于包过滤的访问控制技术，它可以根据设定的条件对经过服务器传输的数据包进行过滤（条件匹配），即对接收到的报文进行匹配和过滤，基于请求报文头部中的源地址、源端口、目标地址、目标端口、请求方法、URL、文件后缀等信息内容进行匹配并执行进一步操作，比如允许其通过或丢弃

​

ACL 是 HAProxy 中的一种条件匹配规则，它可以根据请求中的各种属性（如 IP 地址、URL、HTTP 头部等）来判断请求是否符合特定条件。如果符合条件，则可以将请求转发到指定的服务器组或进行其他操作。ACL 可以与 backend、frontend、listen 等关键字结合使用

​

HAProxy 提供了 ACL（Access Control List）功能，可以根据条件匹配请求并进行转发或拒绝

ACL 条件

其中，<acl-name> 是 ACL 的名称，<condition>是 ACL 的条件，可以是以下类型之一：

  • IP 地址

可以使用 src 或 dst 来匹配请求的源 IP 地址或目标 IP 地址

​

  • 端口号

可以使用 src_port 或 dst_port 来匹配请求的源端口号或目标端口号

​

  • URL

可以使用 path 或 url 来匹配请求的URL

​

  • HTTP 头部

可以使用 hdr( ) 来匹配请求中的 HTTP 头部

​

  • Cookie

可以使用 cook( )、cook_beg( )、cook_end( ) 等函数来匹配请求中的 Cookie

​

​

ACL 匹配模式

在 HAProxy 中，ACL 可以使用多种匹配模式和操作符来实现灵活的匹配和操作，而 ACL-flags 则提供了一些附加的标志，用于进一步控制 ACL 的行为

​

​

ACL 操作符

ACL 可以使用多种匹配模式来匹配请求，以下是 HAProxy ACL 支持的匹配模式：

字符比较：

  1. exact（精确匹配）：指定精确的匹配项。exact abc 表示只有当请求内容为 abc 时才匹配
  2. prefix（前缀匹配）：指定匹配项的前缀。prefix abc 表示只有当请求内容以 abc 开头时才匹配
  3. suffix（后缀匹配）：指定匹配项的后缀。suffix abc 表示只有当请求内容以 abc 结尾时才匹配
  4. regex（正则表达式匹配）：使用正则表达式来匹配请求内容。regex abc.*123 表示只有当请求内容匹配正则表达式 abc.*123 时才匹配
  5. len（长度匹配）：指定匹配项的长度。len 3 表示只有当请求内容长度为3时才匹配
  6. sub（子字符串匹配）：指定匹配项的子字符串。sub abc 表示只有当请求内容中包含子字符串 abc 时才匹配
  7. dom（域名匹配）：指定域名匹配项。dom example.com 表示只有当请求的域名为 example.com 时才匹配
  8. path（路径匹配）：指定路径匹配项。path /index.html 表示只有当请求的路径为 /index.html 时才匹配
  9. hdr（HTTP 头部匹配）：指定 HTTP 头部匹配项。hdr(host) -i example.com 表示只有当请求的 Host 头部为 example.com` 时才匹配

ACL 匹配后操作

匹配后执行的操作

配置 ACL 匹配后执行的操作，实现进一步控制请求的处理流程。匹配成功后，可以执行以下操作之一：

​

范例一：

使用正则表达式匹配 User-Agent 请求头，如果匹配成功则使用 mobile_backend 后端处理请求，否则使用 desktop_backend 处理请求

​

范例二：allow 允许请求通过

  • 当匹配成功后，如果执行 allow 操作，则请求将被允许通过
  • 当请求路径以 /admin 开头时，执行 http-request allow 操作，否则执行 http-request deny 操作

​

范例三：deny 拒绝请求

  • 当匹配成功后，如果执行 deny 操作，则请求将被拒绝
  • 当请求来自 IP 地址为 192.168.1.100 时，执行 http-request deny 操作

​

范例四：http-request redirect 重定向请求到指定的 URL

  • 当匹配成功后，如果执行 http-request redirect 操作，则请求将被重定向到指定的 URL
  • 当请求的 User-Agent 包含 MSIE 6 或 MSIE 7 时，执行 http-request redirect 操作，将请求重定向到 /update-browser.html 页面

​

范例五：use_backend 使用指定的后端处理请求

  • 当匹配成功后，如果执行 use_backend 操作，则请求将由指定的后端处理
  • 当请求路径以 /api 开头时，执行 use_backend 操作，将请求转发到 api_backend 后端处理

​

范例六：http-request set-header 设置请求头

  • 当匹配成功后，如果执行 http-request set-header 操作，则可以设置请求头
  • 当请求的 Host 头为 internal.example.com 时，执行 http-request set-header 操作，设置请求头 X-Internal 为 true

​

范例七：http-request set-path 设置请求路径

  • 当匹配成功后，如果执行 http-request set-path 操作，则可以设置请求路径
  • 当请求路径以 /old-path 开头时，执行 http-request set-path 操作，将请求路径设置为 /new-path

​

范例八：http-request set-method 设置请求方法

  • 当匹配成功后，如果执行 http-request set-method 操作，则可以设置请求方法
  • 当请求方法为 POST 时，执行 http-request set-method 操作，将请求方法设置为 GET

​

​

操作条件

需要注意的是，在使用 if 和 unless 时，只能选择其中一个。使用 if 表示只有当 ACL 匹配成功时才执行操作，而使用 unless 表示只有当 ACL 匹配失败时才执行操作。location 则通常用于 http-request redirect 操作中，用于指定重定向到的目标 URL
