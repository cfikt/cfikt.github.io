HAProxy 配置文件

HAProxy 的配置文件 haproxy.cfg 由两大部分组成，分别是 global 和 proxies 部分

​

​

全局配置段

global：全局配置段

  • 进程与安全配置
  • 性能调整
  • Debug 参数

​

​

代理配置段

proxies：代理配置段

defaults 配置段

listen 配置段

HAProxy 中的 listen 配置段用于定义一个监听器，用于监听客户端的请求，并将请求转发到后端服务器。在 listen 配置段中，可以定义多个后端服务器组，以便进行负载均衡。还可以定义 ACL 规则，根据请求的条件来选择具体的后端服务器组

​

​

server 配置项

server 配置项用于定义后端服务器的相关参数，包括 IP 地址、端口号、权重、最大连接数、健康检查等

  • server_name：后端服务器的名称
  • IP_address：后端服务器的 IP 地址
  • port：后端服务器的端口号
  • options：其他参数选项

listen 实战范例

以上示例定义了两个后端服务器，分别为 web1 和 web2，IP 地址分别为 10.0.0.11、10.0.0.21，端口号为 80，权重为 10，最大连接数为 1000，启用了健康检查功能，健康检查的时间间隔为 5 秒，健康检查成功的次数为 2 次，健康检查失败的次数为 3 次

frontend 配置段

HAProxy 中的 frontend 配置段用于配置前端接收器，即接收客户端请求并将其转发到后端服务器

​

  • 绑定 IP 和端口

frontend 需要绑定一个 IP 地址和端口，以便接收客户端请求。使用 bind 配置项进行配置

表示绑定所有可用 IP 地址的 80 端口

​

  • 关联后端服务器

frontend 需要将接收到的请求转发到后端服务器，使用 default_backend 配置项指定默认的后端服务器

表示将所有请求转发到名为 servers 的后端服务器

​

  • 配置 ACL 规则

ACL（Access Control List）可以用来匹配请求的各种属性，比如请求的 URL、请求的来源 IP 地址等。可以使用 acl 和 use_backend 配置项来配置 ACL 规则

表示对以 /static、/images 和 /javascript 开头的 URL 进行匹配，并将匹配到的请求转发到名为 static_servers 的后端服务器，否则转发到名为 servers 的后端服务器

​

  • 配置 SSL/TLS

frontend 可以配置 SSL/TLS，以支持加密传输。使用 bind 配置项来指定 SSL 证书和私钥的文件路径

表示绑定所有可用 IP 地址的 443 端口，并使用 /etc/ssl/certs/mydomain.pem 文件中的证书和私钥进行 SSL/TLS 加密传输

​

​

backend 配置段

HAProxy 中的 backend 配置段用于配置后端服务器，即接收来自 frontend 的请求并将其转发到后端服务器

​

  • 配置后端服务器

backend 需要配置后端服务器，可以使用 serve r配置项指定一个或多个后端服务器

表示配置了两个后端服务器，分别为 192.168.1.100 和 192.168.1.101，都监听 80 端口

​

  • 配置负载均衡算法

backend 可以使用 balance 配置负载均衡算法

表示使用轮询算法进行负载均衡

​

  • 配置健康检查

backend 可以配置健康检查，以检查后端服务器的可用性。可以使用 check 配置项来启用健康检查

表示对 server1 和 server2 进行健康检查

​

  • 配置超时时间

backend 可以配置超时时间，以控制请求在后端服务器上的处理时间。可以使用 timeout 配置项来配置超时时间

表示设置连接超时时间为 5 秒，服务器超时时间为 30 秒

​

​

​

​

子配置文件

当业务众多时，将所有配置都放在一个配置文件中，会造成维护困难。可以考虑按业务分类，将配置信息拆分，放在不同的子配置文件中，从而达到方便维护的目的

注意：子配置文件的文件后缀必须为 .cfg

​

​

​

​

绑定不存在的 IP

HAProxy 默认不支持配置当前主机不存在的 IP 作为前端访问地址，若想实现此功能，则需要修改内核参数 net.ipv4.ip_nonlocal_bind1 = 1
