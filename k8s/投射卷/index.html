投射卷简介

许多应用服务会从配置文件、命令行参数或者环境变量中读取配置信息。但是容器镜像是只读的，这些配置信息需要与容器解析解耦，但是每次修改一个配置都重新打包一个镜像是不现实的。所以在 Kubernetes 中就引入了 ConfingMap 与 Secret 资源对象。Pod 以存储卷的形式使用 ConfingMap 与 Secret 对象，挂载并加载对象中存储的相关配置信息，降低了配置文件与镜像的耦合关系，提高了镜像的复用能力

Kubernetes 借助了 ConfigMap 和 Secret 资源对象实现了将配置文件从容器镜像中解耦，从而增强了工作负载的可移植性，使其配置更容易更改和管理，并避免了将配置数据硬编码到 Pod 对象的配置清单中，一个镜像加载不同的配置就可以设配不同的环境

ConfigMap & Secret：

  • ConfigMap 与 Secret 都是为容器中的应用提供配置信息的组件，它们把后端真正的数据保存在 API Service 当中，但却能够让 Pod 以存储卷的形式来访问这两种资源类型中对应的数据。而 Pod 想要访问其内部的数据，则需要特殊的存储卷插件 configMap 与 secret，我们需要借助于这两种卷插件去访问其对应资源类型的存储卷
  • API Service 就是系统的配置中心，所有配置的修改都可以直接通过 API Service 完成，由 API Service 接收到配置后，自动下发并注入到所有引用该配置的 Pod 对象，所有引用该配置的 Pod 对象都会收到并自动更新

ConfigMap 简介

说明：ConfigMap 属于名称空间级别的资源，只能被同一名称空间下的 Pod 引用

  • ConfigMap 是一种特殊的存储卷类型，ConfingMap 提供了一种向容器内部注入配置信息的途径，ConfingMap 中的数据可以被 Pod 作为一个数据卷所挂载，在外部重新修改了 ConfigMap 对象中的存储数据后，这些内容都会以近乎实时的方式注入到 Pod 内部中
  • ConfiigMap 将相应的配置信息保存在资源对象中，而后在 Pod 对象中以 confingMap 类型存储卷的形式将其挂载并加载相关配置信息

Secret 简介

说明：Secret 属于名称空间级别的资源，只能被同一名称空间下的 Pod 引用

  • 与 ConfigMap 类似，Secret 也是为容器中的应用提供配置信息的一种特殊存储卷资源对象，不同的是 Secret 主要用于配置敏感信息，比如密码、密钥、令牌等敏感信息。Secret 敏感数据的配置问题
  • Secret 将相应的配置信息保存在资源对象中，而后在 Pod 对象中以 secret 类型存储卷的信息将其挂载并加载相关配置信息

资源配置规范

ConfigMap 和 Secret 资源都是数据承载类的组件，是 Kubernetes API 的标准资源类型类型，是一等公民

ConfigMap 资源规范

ConfigMap 主要负责提供 key-value 键值对格式的数据项，其值支持：

  • 单行字符串：通常用于保存环境变量值，命令行参数等
  • 多行字符串：通常用于保存配置文件的内容

创建 ConfigMap 对象的方法有两种：

  • 命令式

  • 配置文件

    ConfigMap 资源规范中不使用 spec 字段，而是直接使用 data 或 binaryData（二进制格式）嵌套定义 key-value 键值对数据项。一个键值对就对应一个文件，键就是文件名，值就是文件内容

Secret 资源规范

Secret 的类型：

注意：不同类型的 Secret，在定义时支持使用的标准字段也有所不同

除了以下列出的内置类型以外，还可以使用 Opaque 类型中的 —type 选项定义并使用自己 Secret 类型，如果 type 值为空字符串，则被视为 Opaque 类型

内置类型 用法
Opaque 用户定义的任意数据
kubernetes.io/service-account-token 服务账号令牌
kubernetes.io/dockercfg ~/.dockercfg 文件的序列化形式
kubernetes.io/dockerconfigjson ~/.docker/config.json 文件的序列化形式
kubernetes.io/basic-auth 用于基本身份认证的凭据
kubernetes.io/ssh-auth 用于 SSH 身份认证的凭据
kubernetes.io/tls 用于 TLS 客户端或者服务器端的数据
bootstrap.kubernetes.io/token 启动引导令牌数据

Secret 主要负责提供 key-value 键值对格式的数据项，其值支持：

  • 单行字符串：通常用于保存环境变量值，命令行参数等
  • 多行字符串：通常用于保存配置文件的内容

创建 Secret 对象的方法有两种：

  • 命令式：Secret 有类型子命令，不同类型在 data 或 stringData 字段中支持嵌套使用的 Key 有所不同

  • 配置文件

    Secret 资源规范中不使用 spec 字段，而是直接使用 data（Base64 编码格式） 或 stringData（不使用 Base64 编码的明文格式）嵌套定义 key-value 键值对数据项。一个键值对就对应一个文件，键就是文件名，值就是文件内容

引用投射卷

引用 ConfigMap

在 Pod 中引用 ConfigMap 资源对象中保存的键值对数据，通常有两种方式：

  • 环境变量：① valueFrom：以 valueFrom 形式赋值给 Pod 上指定的环境变量，引用 ConfigMap 对象上的特定键值对；② envFrom：以 envFrom 形式一次性将 ConfigMap 对象上的所有键值对导入到 Pod 中。Key 为变量名，Value 为变量值

  • configMap 卷：在 Pod 上将 ConfigMap 对象引用为存储卷，而后将其挂载到容器中的某个目录下。Key 为文件名，Value 为文件内容

引用 Secret

在 Pod 中引用 Secret 资源对象中保存的键值对数据，通常有两种方式：

  • 环境变量：① valueFrom：以 valueFrom 形式赋值给 Pod 上指定的环境变量，引用 Secret 对象上的特定键值对；② envFrom：以 envFrom 形式一次性将 Secret 对象上的所有键值对导入到 Pod 中。Key 为变量名，Value 为变量值

    在 Secret 中，强烈不建议使用环境变量的形式引用配置数据

  • Secret 卷：在 Pod 上将 Secret 对象引用为存储卷，而后将其挂载到容器中的某个目录下。Key 为文件名，Value 为文件内容
